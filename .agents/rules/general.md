# General rules

## General

- DO NOT create new files, folders, or structures without a clear need.
- DO NOT change existing naming conventions.
- DO NOT “improve” code based only on personal preference.
- DO NOT refactor large areas without explicit request.
- ALWAYS follow the existing project standards.

## Build

- DO NOT modify build files without understanding the impact.
- DO NOT change automatically generated files.
- Existing scripts should be reused whenever possible.
